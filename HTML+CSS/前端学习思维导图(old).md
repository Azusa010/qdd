## 💡 六、代码风格建议

基于对你代码的分析，给你几个小建议：

### ✅ 做得好的地方
1. **命名规范**：使用英文类名（header, nav, content 等）
2. **注释清晰**：27-画盒子.html 中有良好的结构注释
3. **代码复用**：使用 .center 类实现居中复用
4. **CSS Reset**：习惯使用 * { margin: 0; padding: 0 }

### 📝 改进建议
1. **缩进统一**：部分文件缩进不一致，建议使用 2或4空格
2. **语义化**：可以多使用语义化标签（如 `<header>`、`<main>`）
3. **CSS 顺序**：建议按"布局 → 盒模型 → 文本 → 外观"顺序写属性
4. **图片 alt**：记得填写有意义的替代文本（不仅是""）
